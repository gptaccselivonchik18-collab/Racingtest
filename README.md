# Neon Asphalt Rush

**Neon Asphalt Rush** is a self-contained 3D arcade racing game for Android built with **Godot 4.x**. It is an offline APK project: no Google Play Services, no ads, no donations, no accounts, no server, no analytics, and no internet permission.

The game is procedural/stylized low-poly 3D: the cars, roads, barriers, buildings, desert props, mountain props, checkpoints, HUD and touch controls are generated by Godot scripts at runtime.

## Downloading the APK

After any push to `main`, GitHub Actions builds the Android APK in the cloud.

1. Open this repository on GitHub.
2. Go to the **Actions** tab.
3. Open the latest successful workflow named **Build Android APK**.
4. Scroll to **Artifacts**.
5. Download **neon-asphalt-rush-apk**.
6. Extract the downloaded ZIP.
7. Install `neon-asphalt-rush.apk` on an Android phone.

The workflow also tries to create a GitHub Release on every push to `main`. If that step succeeds, the APK is also available in **Releases** as `neon-asphalt-rush.apk`.

## Installing on Android

1. Download `neon-asphalt-rush.apk` to the Android phone.
2. Open the file from Downloads or a file manager.
3. Android may show a warning because the APK is not from Google Play.
4. Allow installation from this source.
5. Install the APK.
6. Launch **Neon Asphalt Rush**.

The unknown-source warning is normal for manually installed personal APKs. This game does not request internet access and does not use accounts, ads, analytics, Google Play Services or servers.

## Controls

### Android touch controls

- Left button: steer left
- Right button: steer right
- GAS: accelerate
- BRAKE: brake/reverse
- DRIFT: handbrake/drift
- NITRO: boost
- Pause button: pause menu

### PC test controls

- `W`: gas
- `S`: brake
- `A` / `D`: steer
- `Space`: drift/handbrake
- `Shift`: nitro
- `Esc`: pause

## Screens

- **Main Menu**: Play, Garage, Settings, Quit on desktop platforms.
- **Garage**: car selection, 3D preview, stats, locked/unlocked status, price, buy/select.
- **Track Select**: three tracks with difficulty and best time.
- **Race**: 3D race, HUD, speed, timer, nitro, score, drift score and checkpoints.
- **Pause**: Resume, Restart, Garage, Main Menu.
- **Results**: time, best time, earned money, score, max speed, drift score, next race, garage, menu.

## Cars

1. **Street Starter** — balanced starter car, unlocked immediately.
2. **Sport Viper** — fast acceleration and high speed, worse high-speed control.
3. **Iron Muscle** — heavy stable muscle car, strong launch, worse braking.
4. **Drift Ghost** — best drift control and easy slides, medium top speed.
5. **Hyper Nova** — fastest car and strongest nitro, harder to control.

Each car has parameters that affect gameplay: `max_speed`, `acceleration`, `handling`, `braking`, `drift_control`, `nitro_power`, `mass`, `price`, `unlocked`.

## Tracks

1. **City Night** — neon city with buildings, lamps, barriers and flowing turns.
2. **Desert Highway** — long straights, sand, rocks, cactus-style props and max-speed sections.
3. **Mountain Road** — tighter turns, rocks, mountain mood and harder control.

## Progress and saves

The game works offline and stores local progress in Godot user storage:

- money;
- unlocked cars;
- selected car;
- best times by track;
- settings.

Money is earned after races from finish result, time, checkpoints, speed, score and drift score. Locked cars are bought in Garage.

## Settings

- music volume setting value;
- effects volume setting value;
- steering sensitivity;
- graphics quality: Low / Medium / High;
- reset progress.

The current build keeps audio settings and sound-folder structure ready even if no external audio files are included.

## Build notes

No local Android SDK, Godot or Gradle installation is required on the user's device. GitHub Actions downloads Godot 4.x and Android export templates inside the cloud runner, generates a debug keystore inside the workflow workspace, exports an APK, uploads it as an artifact, and attempts to publish it as a Release asset.
